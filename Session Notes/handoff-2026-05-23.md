## What was decided in the brainstorm

- **Reframe:** not "Etsy backend for Collie" — a full firm-OS for her creative studio. Etsy is one revenue stream within the Hub.
- **Domain:** `farleycreative.com` (Canva-registered, DNS managed inside Canva's UI). Hub on `hub.farleycreative.com` subdomain. Verified Canva supports arbitrary CNAME records on registered domains.
- **Canva upgrade:** NO. Pro tier stays. Connect API requires Enterprise ($20-50K/yr) — doesn't pencil. Route around for Phase 1.
- **Build approach:** we build it for her, she lives in it. NOT solo-launch test (framework too complex for that to be the right test).
- **Tenant-first:** repo is `farley-creative-hub`, not `firm-deploy-template`. FirmDeploy framework gets extracted later from what worked. Per HeyeDeploy pattern-thinking rule + beta-tenant self-framing rule.
- **Two-frame discipline:** never reference FirmDeploy / productization in Farley-facing surfaces. Locked via `feedback_beta_tenant_self_framing.md` (filed same day).
